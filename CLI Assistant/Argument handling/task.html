Description

Quite often, developers are encountered using command line arguments. For example, we can recall the following command npm install -D nodemon, where the -D argument indicates that we want to keep the nodemon package as a development dependency. As another example, consider the npm -h command, where using the -h or --help arguments allows you to list all the commands that you can run using the npm keyword.

We are faced with the task of creating our own CLI, which will add new functionality to us. Regardless of the specific logic of our CLI, we must be able to process command line arguments. For now, we will not specify what exactly we can do with our CLI, let it remain a mystery until the next stage. At this step, you need to create a program that will allow you to process different types of command line arguments and output them in the processed form to the console.

Objectives

Before reading the following rules for processing command line arguments, please take a look at the examples, they can give you more information about the general structure than the text!

In our case, the following rules for processing command line arguments will apply:

  1. If the command line argument is passed with a single dash, then you need to print the value of the argument without a dash and write the value true through the colon, as indicated in the first example below.
  2. If a command line argument is passed with two dashes but no = sign, then it is similarly matched with true. After that, the value of the parameter and the word true should be output to the console through a colon, as in the second example.
  3. If the command line argument is passed using two dashes and simultaneously with a = sign, then instead of the word true, this argument is matched with the value to the right of the equal sign. Check out the third example!
  4. In the event that you passed a command line argument with an equal sign and any number of dashes, but did not give a value to the right of the equal sign, then the following text should be displayed in the console: "It seems you forget to specify argument!". Look at example number four.

Your program must be able to:

  • Receive command line arguments for further processing;
  • Process arguments according to the rules described above;

In order for the second point to become more understandable, you should first look at the examples, and then re-read the task again. Sometimes it's better to show something than to describe it!
