Description

After we were able to process numerous combinations of command line arguments, it is worth moving on to the specific logic of our CLI. Usually, Node.js is presented as a tool to help developers create the backend of sites, which is true. But Node.js also provides a powerful API for working with the file system, of particular interest is the fs (file system) module, which is described in detail in the Node.js documentation.

We will create a CLI that will help us manipulate files using command line arguments, as well as get some data about the status of files / directories.

Objectives

At this stage, our CLI should provide the following functionality:

  1. When passing the --exist=filename argument, the CLI should provide information about the presence or absence of the file. In the first example, the text of the message is indicated, which should be output to the console with a correct and incorrect file name
  2. If the argument --create=foldername is passed, then a folder with the name foldername that you passed as a parameter should be created in the project. The functionality that allows you to create folders and subfolders should also be available, then the argument is as follows --create=foldername/subfoldername. Look at the second example.
  3. If we were able to create folders, then it is logical that we should be able to delete them. Passing the argument --remove=foldername or --remove=foldername/subfoldername should remove folders from the directory, pay attention to the third example
  4. If you want to delete a file that doesn't exist, then you need to print a special message to the console, as shown in the fourth example.
Examples

Example 2: example of output to the console when using the commands node ./main.js --create=project_folder and node ./main.js --create=images/first
