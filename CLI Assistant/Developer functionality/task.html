Description

You have implemented quite a lot of functionality! Let's add even more useful features to make it easier for us to use our CLI assistant.

Suppose we have files with no extension, it would be convenient to check if they are directories or not. It is also quite convenient to find out the size of files in kilobytes without using a graphical tool, and even more convenient to find out the size of not one file, but several files at once! Using the Node.js API, you can make the most diverse functionality, I remind you that all information about all methods can be found in the documentation!

Objectives

At this stage, our CLI should provide the following functionality:

  1. When passing the --isFolder=name argument, the CLI should tell if something called name is a directory or a file, note the first example.
  2. In order to find out the size of one or more files, the CLI must process the --size=first_name-second_name-... argument, where the file names are separated by a -. If you want to know the size of a single file, you can pass the name of a single file as a parameter. The specified functionality is shown in the second example.
  3. If, when using the --isFolder or --size arguments, a folder or file name is passed that does not exist, then the special text specified in the third example should be displayed
  4. The last but not least argument is --content, which prints the contents of the directory to the console, as indicated in the forth example.
Examples

Example 1: example of output to the console when using the commands node ./main.js --isFolder=images and node ./main.js --isFolder=data.txt

Example 2: example of output to the console when using the commands node ./main.js --size=data.txt-other_data.txt and node ./main.js --size=data.txt

Example 3: example of output to the console when using the command node ./main.js --isFolder=fake and node ./main.js --size=fake.txt when fake folder and fake.txt file don't exist

Example 4: example of output to the console when using the command node ./main.js --content
